# Документация Adaspeas

Эта репа хранит и продуктовую документацию, и инженерные инструкции в одном месте, чтобы не плодить бесконечные “ещё один док”.

## С чего начать

1) Продукт и требования (PRD, вижн, флоу): `docs/handbook/01_product/`
2) Архитектура (спеки, контракты, ADR): `docs/handbook/03_arch/`
3) Безопасность и приватность: `docs/handbook/02_security/`
4) Ops (наблюдаемость, runbook, релизы): `docs/handbook/04_ops/`
5) Качество (тест-стратегия, failure modes): `docs/handbook/05_quality/`
6) Планирование (план/вехи/бэклог): `docs/handbook/06_planning/`

## Разработка и деплой

- Контекст проекта и правила работы (включая работу через ChatGPT и команды применения архивов): `docs/CHAT_CONTEXT_RU.md`
- Автоматизация разработки и деплоя (CI/CD, “автоматический режим”): `docs/AUTOMATION_RUNBOOK_RU.md`
- Деплой на VPS: `docs/DEPLOYMENT.md`
- Миграция/перенос на новый VPS: `docs/MIGRATION_RUNBOOK_RU.md`

## Правило, чтобы не плодить новые документы

- Все изменения требований/архитектуры/планов делаются правками существующих файлов в `docs/handbook/`.
- Новые документы допускаются только в двух случаях:
  1) Новый ADR в `docs/handbook/03_arch/ADR/` (решение, которое меняет систему)
  2) Новый Runbook в `docs/handbook/04_ops/` (эксплуатация нового сервиса/риска)


### “Живые” документы и отметки времени

Некоторые файлы являются *точками сохранения* и обязаны иметь понятный таймстемп и короткий “лист регистрации изменений” прямо внутри файла (без отдельного changelog’а):

- `docs/CHAT_CONTEXT_RU.md`
- `docs/AUTOMATION_RUNBOOK_RU.md`
- `docs/DEPLOYMENT.md`
- `docs/MIGRATION_RUNBOOK_RU.md`

Формат:
- в начале файла: `Актуально на: YYYY-MM-DD HH:MM UTC` (или `Updated at:` для англ. файлов)
- в конце файла: `## История изменений` / `## Changes` (таблица: Дата/время UTC, Автор, Тип, Кратко, Причина/ссылка, Commit/PR)

Правило: если правим “живой” документ, обязаны обновить `Актуально на` и добавить строку в таблицу изменений.

Если тянет создать “Doc_v2_final_final.md”, остановись, сделай кофе и обнови существующий раздел.
