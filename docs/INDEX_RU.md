# INDEX (RU): вход для проекта

Актуально на: 2026-02-06 00:10 MSK

Если ты новый чат/ИИ или новый разработчик в проекте:
1) Начни с этого файла.
2) Прочитай **`docs/WORKFLOW_CONTRACT_RU.md`** и следуй ему строго.
3) Дальше выбирай документ по задаче:
   - “Что строим и зачем”: `docs/PRD_RU.md`
   - “Как устроено”: `docs/TECH_SPEC_RU.md`
   - “Как запускать и чинить”: `docs/OPS_RUNBOOK_RU.md`
4) Решения “почему так”: `docs/adr/`
5) Итоги сессий (память чатов): `docs/CHATLOG_RU.md`

Если хочется понять логику структуры документации (а не правила) — `docs/DOCS_RATIONALE_RU.md`.

## Source of Truth (что за что отвечает)

- Процесс, поставка изменений, запреты: `docs/WORKFLOW_CONTRACT_RU.md`
- Продуктовые границы и сценарии: `docs/PRD_RU.md`
- Техническая архитектура и контракты: `docs/TECH_SPEC_RU.md`
- Эксплуатация/инциденты/деплой: `docs/OPS_RUNBOOK_RU.md`
- Приоритеты и этапы: `docs/ROADMAP_RU.md`
- Память по итогам сессий: `docs/CHATLOG_RU.md`
- Архитектурные решения: `docs/adr/`

## Сценарии

- Новый чат/ИИ → INDEX → CONTRACT → по задаче PRD/TECH/OPS.
- Инцидент → OPS “первые 90 секунд” → фиксация в CHATLOG.
- Изменение требований/архитектуры → PRD/TECH (+ ADR при развилке).
- Итоги сессии → запись в CHATLOG.
