# INDEX (RU): вход для нового чата/ИИ

Актуально на: 2026-02-07 17:59 MSK
Если ты новый чат/ИИ для этого проекта:
1) Прочитай **`docs/WORKFLOW_CONTRACT_RU.md`** и следуй ему строго.
   - Напоминание: репозиторий ведём в **одну ветку `main`** (без feature-веток).
   - Если в ответе выдан **инкрементальный pack (.tar.gz)**, в этом же сообщении обязан быть **готовый блок команд** (применение + commit + push) по `docs/PACK_APPLY_TEMPLATE_RU.md`, с уже подставленным именем файла пакета.
2) Дальше выбирай документ по задаче:
   - Продукт и границы: `docs/PRD_RU.md`
   - Технические контракты (reference): `docs/TECH_SPEC_RU.md`
   - Эксплуатация/деплой/перенос/инциденты: `docs/OPS_RUNBOOK_RU.md`
3) Решения “почему так”: `docs/adr/`
4) Итоги сессий (память чатов): `docs/CHATLOG_RU.md`
5) История релизов/важных изменений: `CHANGELOG.md`

## Сценарии
- Новый чат/ИИ → CONTRACT → по задаче PRD/TECH/OPS.
- Инцидент → OPS “90 секунд” → rollback → постмортем.
- Изменение контрактов/поведения → TECH + CHANGELOG (+ ADR если меняется решение/инвариант).
- Итоги сессии → запись в CHATLOG.


## PRE-FLIGHT перед любой задачей
Перед началом любой работы ассистент обязан явно сказать, что нужно: подключить интернет, загрузить архив/pack или подключить GitHub.
Формат и правила см. `docs/WORKFLOW_CONTRACT_RU.md` (раздел 4.1).

## Переезд чата
- Ничего «не переносим текстом» в новый чат. Источник истины — репозиторий.
- В новом чате первая инструкция: «прочитай docs/INDEX_RU.md и следуй docs/WORKFLOW_CONTRACT_RU.md».
- Если GitHub недоступен или есть сомнения в актуальности: загружаем свежий архив репозитория.
