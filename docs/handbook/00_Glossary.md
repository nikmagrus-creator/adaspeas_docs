# Glossary

- **Bot**: приложение, принимающее апдейты Telegram (через Local Bot API) и управляющее UX/каталогом.
- **Worker**: фоновый потребитель очереди, выполняющий скачивание и доставку файлов.
- **Local Bot API**: локальный прокси Telegram Bot API, работающий на VPS.
- **Yandex Disk (YD)**: внешнее хранилище документов; доступ по OAuth.
- **Category**: узел дерева навигации; может содержать подкатегории и файлы.
- **File**: документ в каталоге; хранится в YD, в БД хранится метаданные + yd_path.
- **Job**: единица работы скачивания/доставки файла пользователю.
- **Queue**: транспорт jobs (Redis); не является источником истины.
- **Source of Truth (SoT)**: SQLite как канонический источник состояния и данных.
- **WAL**: Write-Ahead Logging режим SQLite для конкурентного чтения/записи.
- **RBAC**: role-based access control (USER/ADMIN).
- **Invite**: одноразовый код с TTL для активации пользователя/назначения роли.
- **Heartbeat**: периодическое обновление метки времени для обнаружения “зависших” jobs/воркеров.
- **Spool**: краткоживущий файл во временном каталоге как fallback к streaming.
- **SLA/SLO**: целевые показатели (например, навигация < 500ms).
- **RTO**: целевое время восстановления после падения (< 1 мин).
