# DOCS_RATIONALE (RU)

Актуально на: 2026-02-05 03:00 MSK

Этот документ объясняет, почему документация разложена именно так. Нормативные правила (MUST/SHOULD) находятся в `docs/WORKFLOW_CONTRACT_RU.md`.

## Коротко

Здесь цель простая: чтобы через 3 месяца (или 3 человека спустя) было понятно:
- что вообще строим (PRD),
- как оно устроено (TECH),
- как оно живёт в проде (OPS),
- где фиксировать “память проекта” (CHATLOG) и решения (ADR),
- и куда смотреть в первую очередь (INDEX).

## Принцип

Мы стараемся держать документацию компактной и предсказуемой, в духе разделения по назначению:
- **INDEX** — навигация и “куда идти”.
- **WORKFLOW CONTRACT** — правила процесса и поставки изменений.
- **PRD** — цели, границы, пользовательские сценарии.
- **TECH SPEC** — архитектура, контракты, инварианты.
- **OPS RUNBOOK** — запуск/эксплуатация/траблшутинг.
- **CHATLOG** — краткая фиксация итогов сессий.
- **ADR** — отдельные “почему так” по ключевым развилкам.
