# DOCS_RATIONALE (RU): почему структура документации именно такая

Актуально на: 2026-02-05 03:00 MSK
## Коротко
Мы держим документацию компактной и предсказуемой:
- один вход (`docs/INDEX_RU.md`)
- один “протокол работы + память проекта” (`docs/WORKFLOW_CONTRACT_RU.md`)
- три “смысловых” документа (PRD/TECH/OPS)
- CHANGELOG для заметных изменений
- ADR для редких необратимых решений
- CHATLOG для итогов сессий (память чатов без мусора)

## Принцип
Документация разделена по назначению (идея Diátaxis):
- INDEX — навигация и “куда идти”
- OPS — how-to: процедуры “что делать когда”
- TECH — reference: точные контракты/инварианты (MUST/SHOULD)
- PRD — explanation: границы и цели продукта
- ADR — “почему так” для необратимых решений
- CHATLOG — инженерные итоги сессий (не сырой чат)

## Почему не плодим файлы
Дублирующиеся документы неизбежно начинают противоречить друг другу.
Нормативные правила “когда можно/нельзя добавлять новые файлы” и формат поставки изменений закреплены в `docs/WORKFLOW_CONTRACT_RU.md`.
